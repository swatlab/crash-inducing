15290: # This file defines the locations at which this HTTP server may be accessed.
15290: # It is referred to by the following page, so if this file moves, that page must
15290: # be modified accordingly:
15290: #
15290: # http://developer.mozilla.org/en/docs/Mochitest#How_do_I_test_issues_which_only_show_up_when_tests_are_run_across_domains.3F
15290: #
15290: # Empty lines and lines which begin with "#" are ignored and may be used for
15290: # storing comments.  All other lines consist of an origin followed by whitespace
15290: # and a comma-separated list of options (if indeed any options are needed).
15290: #
15290: # The format of an origin is, referring to RFC 2396, a scheme (either "http" or
15290: # "https"), followed by "://", followed by a host, followed by ":", followed by
15290: # a port number.  The colon and port number must be present even if the port
15290: # number is the default for the protocol.
15290: #
15290: # Unrecognized options are ignored.  Recognized options are "primary" and
18848: # "privileged", "nocert", "cert=some_cert_nickname". 
18848: #
18848: # "primary" denotes a location which is the canonical location of
15290: # the server; this location is the one assumed for requests which don't
18848: # otherwise identify a particular origin (e.g. HTTP/1.0 requests).  
18848: #
18848: # "privileged" denotes a location which should have the ability to request 
18848: # elevated privileges; the default is no privileges.
18848: #
18848: # "nocert" makes sense only for https:// hosts and means there is not
18848: # any certificate automatically generated for this host.
18848: #
18848: # "cert=nickname" tells the pgo server to use a particular certificate
18848: # for this host. The certificate is referenced by its nickname that must
18848: # not contain any spaces. The certificate  key files (PKCS12 modules)
18848: # for custom certification are loaded from build/pgo/ssltunnel/certs
18848: # directory. When new certificate is added to this dir pgo/ssltunnel
18848: # must be builded then.
